# PoC(Proof of Concept)

PoC는 **Proof of Concept(개념검증)** 입니다.
즉, “이 설계/기술 선택이 실제로 **되는지**를 작은 범위로 먼저 만들어서 **검증**하는 단계입니다.

## PoC의 핵심 목적

* **기술 선택 확정**: HtmlUnit으로 브라우저 시나리오가 충분히 안정적으로 되는지 확인
* **리스크 조기 발견**: SPA/로그인/동적 로딩에서 오탐·타임아웃이 나는지 확인
* **운영 비용 추정**: 실행 시간, 실패율, 리소스 부담(특히 Puppeteer 대비)을 수치로 확인

## 이번 과제에서 PoC가 의미하는 것

* 브라우저/모듈 시나리오를 “풀스펙”으로 만들기 전에,
* **대표 시나리오 3개(정적/로그인/SPA)** 를
* HtmlUnit로 먼저 구현해서 **N회 반복 실행**하고
* 성공률/오탐률/평균 응답시간/실패 원인을 기록한 다음
* 그 결과로 “HtmlUnit로 간다 / 일부는 Puppeteer 필요”를 결정하는 것

## PoC vs MVP 차이

* **PoC**: “기술적으로 가능한가/안정적인가” 검증(작고 빠르게)
* **MVP**: “사용자가 쓸 수 있는 최소 제품” (기능 완성도/UX 포함)
